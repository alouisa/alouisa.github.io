Ch9: The Window Object

  • Global Object
    • If you don’t know the name of the global object, you can also refer to it using the keyword this in the global scope
      • const global = this;
    • use window.variableName if you need to check whether a global variable has been defined (i.e. in if())
  • window dialogs
    • alert()
    • confirm()
    • prompt()
    • these methods will stop the execution of a program in its tracks
    • It’s also worth keeping in mind that most browsers allow users to disable any dialogs from repeatedly appearing, meaning they are not a feature to be relied upon.
  • Browser Information
    • navigator property
      • .userAgent
        • Don’t rely on this information though, as it can be modified by a user to masquerade as a different browser.
        • every browser will include the string “Mozilla” in its userAgent property
  • location property
    • .href
    • .protocol
    • .host
    • .hostname
    • .port
    • .pathname
    • .search
    • .hash
    • .origin

    • .reload()
    • .assign()
    • .replace()
    • .toString()
  • The Browser History
    • window.history
      • .length
      • .go()
      • .forward()
      • .back()
  • Controlling Windows
    • window.open()
    • .close()
      • can be used to close a window, assuming you have a reference to it (i.e. popup.close())
    • .moveTo()
    • .resizeTo()
    • It’s a bad idea from a usability standpoint to resize or move a user’s window
  • Screen Information
    • window.screen
      • .height
      • .width
      • .availHeight
      • .availWidth
      • .colorDepth
      • The Screen object has more uses for mobile devices. It also allows you to do things like turn off the device’s screen, detect a change in its orientation or lock it in a specific orientation.
  • The Document Object
    • This object has properties and methods that deal with the page that has been loaded into the window
    • document
      • .write()
        • heavily frowned upon as it can only be realistically used by mixing JavaScript within an HTML document
  • Cookies
    • small files that are saved locally on a user’s computer
    • stores info that can then be retrieved between requests
    • Their use for data storage is starting to be replaced in many cases by the new HTML5 localStorage API as it allows more data to be stored.
    • Cookies are still useful for retaining state information (such as if a user is logged in) because they’re passed between the client and server on every HTTP request.
    • document.cookie
      • Changing cookie value
        • A cookie’s value can be changed by reassigning it to document.cookie using the same name but a different value
      • Reading Cookies
        • const cookies = document.cookie.split("; "); for (crumb of cookies){ const [key,value] = crumb.split("="); console.log(`The value of ${key} is ${value}`); }
      • Cookie Expiry Dates
        • "expires=date"
        • "max-age=numInSecs"
      • The Path and Domain of Cookies
        • "path=/"
        • "domain=domainName"
      • Secure Cookies
        • 'secure'
      • Deleting Cookies
        • Set expiration date to past
      • Libraries
        • Cookies.js
        • sCookie
  • Timing Functions
    • window
      • .setTimeout()
      • .clearTimeout()

      • .clearInterval()
