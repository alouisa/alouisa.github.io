Ch11: Further Functions

  • Functions are first-class objects
  • .length
    • returns # parameters
  • call()
    • set value of this inside a function to an obj provided as 1st arg
  • apply()
    • works in same way as call(), except argument(s) of function as array
  • .description
    • allows to describes what func does
  • memoization
    • "result caching"
    • cache property
  • Immediately Invoked Function Expression– IIFE
    • () at end to envoke func immediately
    • temp
      • uses IIFE
      • not needed in ES6 for swapping
        • let [a,b] = [1,2]; [a,b] = [b,a];
    • can be used to set up any initialization code that there’ll be no need for again
      • can set up any variables, objects and event handlers when the page loads.
      • much of this can be achieved in ES6 by simply placing the code inside a block
    • recommended way to use strict mode is to place all your code inside an IIFE
    • can be used to enclose a block of code inside its own private scope
  • Functions that Define and Rewrite Themselves
    • Ex/ function party(){ console.log('Wow this is amazing!'); party = function(){ console.log('Been there, got the T-Shirt'); } }
  • Init-Time Branching
    • we can define a function based on whether certain methods are supported
    • function is invoked twice the first time, although it becomes more efficient each subsequent time it’s invoked
  • Recursive Functions
    • invokes itself until a certain condition is met
  • Event-driven Asynchronous Programming
    • Instead of waiting for an event to occur, a callback can be created that’s invoked when the event happens
    • event-loop
    • a callback always has to wait for the current execution stack to complete before it’s invoked
    • Callback Hell
      • too many callbacks in same func
  • promise represents the future result of an asynchronous operation
    • pending
      • unsettled
    • settled
      • Resolved
      • Rejected
    • Creating A Promise
      • constructor function
      • func called executor as arg
      • also accepts resolve() or reject() as args
    • Settled Promise
      • .then()
      • .catch()
    • Chaining Multiple Promises
  • Async Functions
    • async keyword
    • await operator
  • Generalized Functions
  • Functions That Return Functions
  • Closures
  • Function Scope
    • closure
      • when the inner function is returned by the outer function, maintaining access to any variables declared inside the enclosing function.
      • variable kept alive outside of its scope
      • whenever a function is defined inside another function, the inner function will have access to any variables that are declared in the outer function's scope
  • Generators
    • function*
      • next()
      • yield
  • Functional Programming
    • Pure Functions
      • MUST HAVE:
        • return value only depend on the values provided as arguments.
        • doesn't change any values or data elsewhere in the program
        • Given the same arguments, a pure function will always return the same result
        • At least one argument
        • return value
        • perform a single task
      • Higher-Order Functions
        • :functions that accept another function as an argument, or return another function as a result, or both
  • Currying
    • involves the partial application of functions
