Master Mobile UX Notes:

  • Design for smallest interface first and build up rather than build down.
  • User interface constantly changing. Current shift is towards mobile devices.
  • Current mobile device per person is 2-3
  • Responsive retrofitting - Building down instead of up - isn't the same as mobile first
    • Cheaper and faster ✔
    • Better than desktop only design ✔
    • Screen real estate problems ✘
    • Touch interaction issues not addressed ✘
  • Mobile First
    • Time consuming and costly to make total rebuild ✘
    • Best experience across devices ✔
    • Addresses mobile size/bandwidth issues ✔
    • Ready for future ✔
  • "A lot of times people don't know what they want until you show it to them."
    • Be there
    • Be useful- provide what they came for as quickly as possible
    • Be quick- keep steps simple and seamless
  • Reachability mobile: 75% rely on thumbs, 49% one handed grip
    • best reachability normally towards bottom of device
  • One of the biggest thing that affects speed is network - not everyone has latest and greatest
  • Ensure things are loading quickly
    • Reduce images and when needed use more svg/webfonts
    • Optimize and minify CS and JS files
    • GZIP files
  • Primary Controls
    • Example: placing most commonly used menu items on bottom
    • Example: adding word menu and border instead of Hamburger menu
  • Forms/Mobile Inputs
    • Try changing labels to float labels
    • Make things like checkboxes larger so they are more reachable
    • Have form change based on responses so only what is needed it displayed
    • Make labels clear and concise
    • Use pagination with progress indicator when form must be long
    • Enable auto complete on a form
    • Forgot password link & Show password by default but allow to HIDE
    • Provide best keybord for input being entered
    • Use anchor tags to link things like phone numbers to call
  • Advertising
    • pop-up/ pop-overs = more detested on web
    • Made ads small and simple- can enlarge once quicked
    • Window below content with full screen ad so not as in your face
    • Keep it appropriate
    • content First
